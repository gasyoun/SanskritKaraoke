# Сценарии использования — Sanskrit Karaoke

Этот документ описывает основные рабочие процессы для различных ролей в экосистеме Sanskrit Karaoke.

---

## 1. Создатель контента (Учитель / Исследователь)
**Цель:** Превратить сырой санскритский текст и аудиозапись в структурированный интерактивный урок для библиотеки.

### Рабочий процесс:
1.  **Ввод текста:** Откройте `index.html`. Вставьте первую половину шлоки (s1) и вторую (s2). Инструмент автоматически определит метр и построит начальную волновую диаграмму.
2.  **Интеграция аудио:** Загрузите запись в формате `.mp3` или `.wav`.
3.  **Корректировка метрики:** Нажмите правой кнопкой мыши на слоги, чтобы исправить их вес (Гуру/Лагху), если автоматика пропустила поэтический нюанс (например, *muta cum liquida*).
4.  **Синхронизация тайминга:**
    *   Откройте **Редактор тайминга**.
    *   Используйте **Режим пад**, чтобы расставить границы 8 пад.
    *   Переключитесь в **Режим слогов**, чтобы точно настроить появление «точки» для каждого слога, используя горячие клавиши (`Ctrl+Arrows`).
5.  **Метаданные и экспорт:**
    *   Заполните форму **Экспорт в библиотеку** (добавьте переводы, уровень сложности и теги).
    *   Скачайте файл `[id].json` и сохраните его в `verses/data/`.
    *   Экспортируйте **PNG** высокого разрешения для печати и **Karaoke MP4** для соцсетей.
6.  **Облачное сохранение:** Сохраните сессию на Google Drive, чтобы иметь возможность внести правки позже.

---

## 2. Активный студент (Ученик)
**Goal:** Системно выучить шлоку и освоить её метрический ритм с помощью интервальных повторений.

### Рабочий процесс:
1.  **Поиск:** Просмотрите `catalogue.html` (или откройте через Telegram Mini App). Отфильтруйте по сложности или метру.
2.  **Учебная сессия:**
    *   **Фаза А (Полный режим):** Слушайте аудио, глядя на подписи слогов, чтобы соотнести звуки с текстом.
    *   **Фаза Б (Режим точек):** Скройте текст. Следуйте за визуальной пульсацией метра (пики волны), напевая шлоку вслух.
    *   **Фаза В (Вслепую):** Всё скрыто, кроме движущегося маркера. Попытайтесь полностью воспроизвести шлоку по памяти.
3.  **Самопроверка:**
    *   Пройдите **Квизы** (определите метр, впишите скрытый слог или отстучите ритм).
    *   Оцените качество своего запоминания (😊/😐/😕).
4.  **Облачная синхронизация:** Войдите в систему через кнопку **Cloud Sync**, чтобы ваш прогресс сохранился в Firebase.

---

## 3. Куратор (Оператор конвейера / Инженер)
**Цель:** Поддерживать качество библиотеки шлок и мониторить эффективность платформы.

### Рабочий процесс:
1.  **Прием контента:** Проверьте Pull Request с новыми JSON-файлами шлок.
2.  **Автоматическая валидация:** Запускается **Teaching Pipeline**:
    *   **VerseCurator** проверяет соответствие схеме.
    *   **ContentEnricher** (через Gemini) заполняет недостающие переводы на русский и английский.
    *   **QualityGate** гарантирует, что название метра соответствует метрической структуре.
3.  **Контроль показателей:**
    *   Запустите `python tools/cost_dashboard.py`, чтобы убедиться, что автоматическое обогащение уложилось в бюджет **$0.10 на шлоку**.
    *   Запустите `python tools/student_stats.py`, чтобы выявить шлоки, которые оказались сложнее ожидаемого (по статистике ошибок студентов).
4.  **Бенчмаркинг:** Запустите `python evals/judge.py`, чтобы убедиться, что переводы LLM не ухудшились после изменений в конвейере.

---

## 4. Оффлайн-практик (Мобильный пользователь)
**Цель:** Продолжать обучение в дороге или в местах с плохой связью.

### Рабочий процесс:
1.  **Подготовка:** Откройте шлоку один раз, пока вы онлайн. Система автоматически кэширует данные сессии.
2.  **Использование оффлайн:** Откройте `student.html?id=[id]`. **Service Worker** предоставит интерфейс, а **Drive Fallback** загрузит данные шлоки из локального кэша.
3.  **Индикатор:** В заголовке появится надпись **«Оффлайн-режим (кэш)»**, подтверждающая использование локальных данных.
4.  **Синхронизация:** Когда связь восстановится, посетите `progress.html`, и ваши результаты оффлайн-практики будут синхронизированы с облаком Firebase.
