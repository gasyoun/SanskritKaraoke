## 🧠 Dev Notes & Hypotheses

### Alignment contract locked (18-07-2026, MG + Codex GPT-5)
- One red-dot transition per syllable; mātrā weights guide duration but never create a second transition inside guru.
- Timestamp target = earliest audible onset (initial consonant, or vowel for vowel-initial syllables). A vowel nucleus may be an internal detection anchor only.
- Real pauses clear the dot. No author-supplied sūtra boundaries; śloka pādas come from the existing Taylor/chandas machinery. Scope includes śloka, prose sūtra, and Vedic material, mainly śloka.
- Approved `sample.mp4` is the behavioral gold. Review measurement: current CLI on its 35 transitions achieved 1/35 within ±50 ms, 448 ms mean error, 1.772 s max, despite confidence 0.657; therefore ADR-0003's algorithm choice is superseded by [ADR-0004](docs/adr/0004-approved-timing-corpus-alignment.md).
- The existing library's pre-human vs post-human JSON pairs are the learning/evaluation corpus. Human approval time under one minute is the binding product gate.
- Shared-main guard event: the documentation commit was blocked in the shared `main` checkout even though repo WIP was empty and `main == origin/main`; the exact source-only delta was re-landed from fresh `origin/main` in worktree branch `codex/alignment-contract-docs`. The untracked user-owned `sample.mp4` stayed in the original checkout and was not copied or staged.

### Student-page integration traps (2026-07-03, Fable 5 `claude-fable-5`) — all fixed, keep in mind for new pages
- app.js is an ES module: any page loading it needs `type="module"` AND every symbol its inline scripts call must be in app.js's window-export block (`student.html` needed `window._applySession`).
- `render()` clears `#main-area.innerHTML` on every pipeline run — page-specific UI must live OUTSIDE it (student panels are in sibling `#student-panels`).
- Programmatic `.value =` fires no input event → the debounced scheme auto-detect never runs; call `forceSchemeDetect()` explicitly before `runPipeline()`.
- `verse.translation` mixes language keys with `provenance`/`rights` metadata — filter to non-empty strings before treating keys as languages.
- Template↔deployed drift is the recurring failure mode of the `make_student.py` byte-copy design: EVERY student.html fix must land in `tools/templates/student.html`, never the deployed file.
- GitHub Pages deploys flake (`"Deployment failed, try again later"`); rerunning only the failed job creates a duplicate `github-pages` artifact and fails again — always dispatch a fresh full run (`gh workflow run pages.yml`).

### Architecture findings (2026-06-13)
- `document.addEventListener('click', closeCtx)` was inside `render()` — accumulated a listener per keystroke. Fixed: moved to module-init (once).
- `_mainHighlightStart()` RAF guard (`if (!_mainHlRaf)`) was already correct — review agent flagged a false positive.
- `openTimingEditor` / `closeTimingEditor` listener pairing is symmetric and correct — no leak. The timing-editor concern in the review was a false positive.
- `_getWaveColors()` is now a named function; headless renderers can shadow it without touching `buildWaveSVG()`.
- `runPipeline()` now has a try/catch; uncaught meter-detection / syllabification throws surface as `showMsg` instead of locking UI.

### Data/schema findings (2026-06-13)
- `TAP.confidence` is populated **only** by the whisper-v1 aligner (`teAutoTimingAndSwitch`), never by manual timing edits. Documented in CLAUDE.md.
- `arrow` field on DATA syllables is serialized in session JSON but absent in verse JSON files. Now documented in CLAUDE.md.
- `vipulaType` known variants: `ma-vipulā`, `bha-vipulā`, `na-vipulā`, `ra-vipulā`. Documented.
- `duration_s: 15.5` was a template sentinel in all 3 sample verses; replaced with `null` and schema updated to `["number","null"]`.

### Security findings (2026-06-13)
- Drive `_gdRenderList()` XSS: `f.name` was embedded in onclick= HTML attribute with only `'` escaping — backslash injection bypassed it. Fixed with `createElement` + `addEventListener`.
- `showMsg()` was missing `>` and `"` from its escape chain — potential XSS if caller ever passes external content.
- Firebase/Google API keys in client code are public-by-design — security is via Firestore rules + authorized domains, NOT key secrecy.
