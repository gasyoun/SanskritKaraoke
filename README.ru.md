Sanskrit Karaoke превращает санскритскую шлоку в интерактивную волновую диаграмму, которая показывает метрический вес каждого слога (гуру / лагху), позволяет задать тайминг по аудио и экспортировать карaoке-видео или изображения высокого разрешения.

---

## Возможности

- **Волновая диаграмма** — каждый слог отображается кружком на одной из пяти горизонтальных строк; тяжёлые слоги (гуру) — тёмно-красные, лёгкие (лагху) — тёмно-зелёные; позиции можно перетаскивать
- **Поддержка кодировок** — ввод текста в деванагари, IAST, SLP1, Harvard-Kyoto, ITRANS, Velthuis и WX; кодировка определяется автоматически
- **Определение метра** — автоматически распознаёт метр (ануштубх, самавритта, джати и другие), отмечает випулы в нечётных падах ануштубха
- **Редактор тайминга** — два режима:
  - *Разметка пад*: 8 линий-границ для грубой разметки начала и конца каждой пады
  - *Тайминг слогов*: точная настройка тайминга для каждого слога с горячими клавишами
- **Экспорт карaoке MP4** — рендер диаграммы с аудиодорожкой и анимированным маркером слога
- **Экспорт PNG** — изображение 1920 × 1080 с деванагари, транслитерацией IAST и метаданными
- **Google Drive** — сохранение и загрузка сессий (JSON + аудио) в общую папку Drive

## Как пользоваться

1. Откройте [приложение](https://gasyoun.github.io/SanskritKaraoke).
2. Вставьте первую и вторую половину шлоки в поля s1 / s2.
3. Нажмите **Run** — диаграмма построится, метр определится автоматически.
4. Тип и положение слогов можно изменить правой кнопкой мыши.
5. Загрузите аудиофайл и откройте **Редактор тайминга**:
   - Сначала используйте режим пад для грубой разметки.
   - Переключитесь в режим слогов для точной настройки.
6. Экспортируйте через кнопки **PNG** или **Каraoke MP4**.

### Горячие клавиши (режим тайминга слогов)

| Клавиша | Действие |
|---|---|
| `←` / `→` | Предыдущий / следующий слог |
| `Ctrl+←` / `Ctrl+→` | Сдвинуть слог на −0.01 с / +0.01 с |
| `Home` | Выровнять к началу пады |
| `Enter` | Проиграть текущий слог |
| `Space` | Проиграть и перейти к следующему |
| `Ctrl+Space` | До конца шлоки |
| `Ctrl+Enter` | До конца пады |

## Дорожная карта

### ~~Экспорт для Telegram Stories~~ ✓ реализовано в v1.267
Портретное видео (9:16 или 1:1): камера плавно следует за маркером активного слога, удерживая в кадре 2–4 слога. Формат и количество слогов задаются в боковой панели. Аудио встроено, 24 кадра/с.

### ~~Библиотека шлок и JSON-схема~~ ✓ реализовано
Структурированный файловый каталог (`verses/data/*.json`) с метаданными шлоки: источник, метр, сложность, переводы (RU/EN), ссылки Drive на аудио и сессию. Включает JSON-схему, скрипт валидации и автогенерацию индекса.

### ~~Страница студенческого плеера~~ ✓ реализовано
Отдельная страница только для чтения (`student.html?id=…`), которая загружает шлоку из каталога, отрисовывает волновую диаграмму и воспроизводит карaoке — без инструментов редактирования.

### ~~Режим прогрессивного раскрытия~~ ✓ реализовано
Кнопка переключения режимов в студенческом плеере: **Полный** (все подписи) → **Только кружки** (без текста) → **Вслепую** (только аудио и маркер).

### ~~Переключение RU / EN i18n~~ ✓ реализовано
Кнопка-глобус в заголовке переключает весь интерфейс между русским и английским языками. Строки вынесены в `src/scripts/strings.js`.

### ~~Интервальные повторения (SM-2)~~ ✓ реализовано
После проигрывания шлоки студент оценивает качество запоминания (😊 😐 😕). Алгоритм SM-2 планирует повторения в оптимальные интервалы. Состояние в localStorage; очередь «Изучать сегодня».

### ~~Квизы для самопроверки~~ ✓ реализовано
Три типа квизов после шлоки: тэппинг гуру/лагху (beat tap), выбор метра из списка, вписывание скрытого слога до того, как его достигнет маркер.

### ~~Трекинг прогресса и страйков~~ ✓ реализовано
Ежедневный страйк, счётчик выученных шлок (интервал SRS ≥ 21 день) и страница `progress.html` со списком всех шлок каталога и их статусами.

### Восстановление режима тэппинга
Тэппинг в реальном времени для быстрой черновой разметки — кнопка скрыта, пока функция дорабатывается.

### Исправление замены файла на Drive
При повторном сохранении сессии старый файл может оставаться на Drive. Требуется последовательность удаление → загрузка в рамках текущего OAuth-скоупа.

### Кросс-проверка по просодии Апте
Завершить модальное окно кросс-проверки: сравнение определённого метра с базой данных Апте и подсветка расхождений.

### ~~Поддержка мобильных устройств и сенсорного ввода~~ ✓ реализовано в v1.2.2
Полное обновление студенческого интерфейса для мобильных устройств. Нижняя панель управления, сворачивающиеся переводы и крупные кнопки для квизов.

### ~~Поддержка Telegram Mini App~~ ✓ реализовано в v1.4.0
Интеграция с Telegram Web App SDK. Синхронизация темной темы, поддержка нативной кнопки «Назад» и автоматическое развертывание.

## Сценарии использования

### 1. Подготовка нового урока (Учитель)
*   **Цель**: Создать качественное караоке-видео и запись в библиотеке для новой шлоки.
*   **Процесс**:
    1.  Вставьте текст в `index.html`. Система определит метр и построит волну.
    2.  Загрузите аудио и используйте **Редактор тайминга**, чтобы синхронизировать маркер.
    3.  Заполните форму **Экспорт в библиотеку** (переводы, теги, сложность).
    4.  Скачайте JSON и добавьте его в репозиторий; CI проверит его и обновит каталог.
    5.  Экспортируйте **Karaoke MP4** для публикации в YouTube или Telegram.

### 2. Ежедневная практика и заучивание (Студент)
*   **Цель**: Выучить шлоку, используя визуальные и звуковые подсказки.
*   **Процесс**:
    1.  Откройте `catalogue.html` в Telegram. Отфильтруйте по тегу или сложности.
    2.  Начните в **Полном режиме**, чтобы выучить слоги и ритм.
    3.  Переключитесь в режим **Только точки**, чтобы тренировать память, видя пульсацию метра.
    4.  Завершите в режиме **Вслепую** (только аудио), чтобы проверить полное запоминание.
    5.  Оцените себя, чтобы обновить расписание **интервальных повторений (SM-2)**.

### 3. Контроль прогресса (Исследователь)
*   **Цель**: Управлять большой библиотекой выученных шлок на протяжении месяцев.
*   **Процесс**:
    1.  Ежедневно проверяйте `progress.html` на наличие шлок в очереди «Изучать сегодня».
    2.  Следите за **страйком** и количеством «выученных» шлок (интервал повторения > 21 дня).
    3.  Используйте статусы в каталоге (Новая / Учу / Выучена) для планирования занятий.
